# Solutions

One Markdown file per solved problem, written by `compound` after a verified
fix or decision earns capture.
Each captures: the symptom, the real root cause, the fix, and how it was verified —
so the next agent (or the next you) doesn't rediscover it from scratch.

This is the payload of the whole system's "compound" step: the corpus a future
`research` / planning pass reads back in. Keep entries specific and evidence-backed;
prune stale ones as the code moves.
