# OpenWiki brief for qq

This file is the standing, operator-owned brief for OpenWiki runs against this
Repository. Read it for scope and priorities; never rewrite it during routine
init/update runs.

## What this wiki is for

Agent-facing operational knowledge of the landed qq system, for fresh-context
Actors who must act correctly without the conversation that produced the code.
The operator reads it second; agents read it first.

## Scope and priorities

- Document the landed system only: merged `main` is the sole source of truth.
  Never document in-flight branches, uncommitted work, or intentions.
- Keep the current lean page shape (quickstart, architecture, workflows,
  skills, verification, operations). Prefer updating existing pages over
  adding pages; add a page only when an existing one would become incoherent.
- Use the capitalized entity vocabulary from `CONCEPTS.md` (Actor, Repository,
  Task, Change, Check, Skill, Knowledge item) consistently.
- Concise operational synthesis of the canonical sources (`README.md`,
  `CONCEPTS.md`, `AGENTS.md`, `skills/*/SKILL.md`) with links back to them is
  exactly what this wiki is for — keep it. What it must not do is reproduce
  their content verbatim or become a competing authority: when a synthesis
  and its source disagree, the source is right and the wiki page is stale.
- Emphasize what an agent must get right: Skill triggers and ownership
  boundaries, the delegation automation contract, verification expectations,
  and operating procedures for the cockpit tools under `bin/`.
- Remove stale content aggressively; a wrong page is worse than a missing one.

## Diagrams

BPMN process diagrams are governed by the authoring extension supplied on each
run by `bin/qq-openwiki`; it carries the machine-specific publisher commands.
This brief does not duplicate those instructions. Follow them as given.

## Boundaries

- All authored content stays under `openwiki/`.
- Do not create or edit repository `AGENTS.md` or `CLAUDE.md`; qq owns those.
- Do not edit this file unless the operator explicitly asks to change the
  brief.
