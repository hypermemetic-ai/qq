# Linear work card template

Use for the rented operator work-surface trial. Linear may own lifecycle and
pending cards; durable engineering facts must still land in repo files, GitHub,
or ledgers.

## Title

`U-YYYYMMDD-short-slug — short work title`

## Body

**Unit:** `units/U-YYYYMMDD-short-slug.md`; inline drafting is allowed only before dispatch  
**Lifecycle:** ready | running | blocked | done  
**Class:** feature | bugfix | refactor | research  
**Repo:** owner/repo  
**Branch/PR:** link once known  
**CI:** link once known  
**Session/run:** Warp/Oz link once known  
**Relevant docs:** policy, memory, unit, acceptance, design note, or adoption record section

### Intent

At most fifteen lines. Operator-authored.

### Acceptance

One shell fenced block of commands/probes CI can run. Operator-authored.

### Floor / blocked decision

Only when needed. Include options, delay cost, machine lean, and flip condition.

### Close condition

The card closes only after the result is reflected in the correct durable home:
unit file, memory file, autonomy ledger, loss ledger, PR, or CI history.
