# Quarterly shrink review

One hour. Delete first.

- [ ] Read losses since last review; no system change without `LL-<n>`.
- [ ] Check mechanism count and glue line count; trend should be flat or down.
- [ ] Delete or demote any verifier rung with zero unique catches this quarter.
- [ ] Autonomy: demote classes with clean escalation/audit history; promote
      audited classes with regret over 1 in 10.
- [ ] Expertise: any important class with repeated `could_explain_cold=no`
      becomes explanation-required or design-first until clean.
- [ ] Operator surface: no shadow queue, hidden vendor-chat decisions, stale
      session links, or unresolved cards remain; any decision handled outside
      the surface because another tool was easier is evidence for or against
      the rental.
- [ ] Memory: promote only useful candidates; delete the rest; keep `memory.md`
      under 200 lines.
- [ ] Gate bypasses, listed first: each needs a written reason or a loss entry.
- [ ] Rentability watch: delete what the market now maintains; flag rentals
      that became unstable, expensive, or hidden authority.
- [ ] Confirm `record/adoption.md` is still frozen; new rationale belongs in
      the loss ledger.
